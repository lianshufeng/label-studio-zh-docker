# 字典扩充参考

## 优先级

1. GitHub 最新正式版源码中的模板、组件、页面、菜单、标题、按钮、说明、备注、模板分类和模板标题
2. 容器运行页实际 HTML
3. Django 模板
4. 常见导航、按钮、表格、弹窗、设置项固定词汇

## 重要约束

1. 字典必须以对应版本源码提取结果为主。
2. 必须优先做全量源码扫描，不能只抽少量高频词。
3. 运行页面巡检只用于补漏和校验，不作为主要词典来源。
4. 常见词汇表只能作为补充，不能替代源码扫描。
5. 若源码中存在字符串，则优先保留源码原文作为映射 key。

## 翻译边界

1. 必翻：标题、按钮、菜单、说明、提示、空状态、帮助文案、弹窗文案、表单标签。
2. 条件翻译：官方内置模板示例内容、固定演示文案、非用户输入的预览说明。
3. 默认不翻：真实任务数据、标注对象原文、字段值、动态业务内容、可能影响标注语义的原始文本。
4. 若无法可靠区分“官方示例”和“真实任务数据”，优先保持不翻，避免污染标注对象。

## 建议覆盖模块

- 登录与注册
- 首页欢迎区与资源卡片
- 项目列表与创建项目
- 导入、导出、云存储
- 数据管理
- 标注页通用按钮
- 标题、按钮、说明、备注、帮助信息、空状态文案
- 成员、角色、组织
- 设置页与危险操作区
- 模板分类
- 模板标题

## 字典维护原则

1. 先补精确短语，不先做模糊替换。
2. 对极短词汇保持克制，避免误替换业务数据。
3. 优先翻译源码中真实出现的按钮、标题、菜单、表单标签、提示文案。
4. 说明文案、备注文案、帮助信息、卡片说明和空状态文案也属于必提取范围。
5. 不采用纯人工页面巡检补词或凭经验整理常见词作为主方法，优先从对应版本源码全量提取。
6. 每次扩充后，至少回看登录页、一个主应用页面和创建项目模板弹窗。
